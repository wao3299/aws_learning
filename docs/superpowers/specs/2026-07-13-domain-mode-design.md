# 分野別出題モード 設計

日付: 2026-07-13

## 目的

統計には分野別正答率が表示されるのに、特定の分野だけを練習する手段がない
ギャップを埋める。

## 挙動

- ホームのモードボタン群に「分野を選んで練習」ボタンを追加する
- 押すと**中央オーバーレイのモーダル**が開き、分野一覧を表示する
  - 分野は問題データ（`ALL` の `d`）から全列挙する（未学習の分野も出す）
  - 各分野に問題数を表示し、学習済みの分野には通算正答率を併記する
  - ✕ ボタン・背景クリック・ESC キーで閉じられる
- 分野を選ぶとモーダルを閉じ、その分野から `practiceN` 問ランダム出題を開始する
  - 分野の問題数が `practiceN` 未満なら全問出題する
- モードタグに「分野別: <分野名>」を表示する
- 統計への記録は通常セッションと同じ（キーは `d|t` のままで変更なし）

## 実装方針

- モーダルはネイティブ `<dialog>` 要素を使う（`showModal()` で
  ESC 閉じ・フォーカストラップが標準で得られる）
- 分野一覧は `ALL` と `STATS` から導出する
- 分野選択時は `startSession(shuffle(該当問題).slice(0, PRACTICE_N), "分野別: <分野名>")`

## 変更ファイル

| ファイル | 変更 |
|---|---|
| `src/scripts/quiz.js` | 分野モーダル描画・開閉、分野別セッション開始 |
| `src/components/Quiz.astro` | 「分野を選んで練習」モードボタン、`<dialog id="domaindlg">` 骨組み |
| `src/styles/quiz.css` | dialog・分野行・バックドロップのスタイル |

## 変更しないもの

- 問題データ形式（`public/questions-*.json`）
- localStorage キー・統計スキーマ（`quizStats_<code>_v1` / `quizHistory_<code>_v1`）
- Firebase 同期（`sync.js` / `merge.js`）— 統計の書き込み経路が変わらないため
- `scripts/validate.js`

## エラーハンドリング

- 分野が 1 つしかないデックでもモーダルは同様に動く（特別扱いしない）
- `<dialog>` 非対応ブラウザは考慮しない（対象ブラウザは最新の Chrome/Safari/Firefox）

## 検証

- `bun run build` が通ること
- `bun run dev` で以下を手動確認する
  - 分野モーダルの開閉（✕・背景・ESC）と分野別出題の開始
  - 分野の問題数 < practiceN のとき全問出題になること
  - 未学習の分野もモーダルに表示されること
  - モードタグに「分野別: <分野名>」が表示されること
